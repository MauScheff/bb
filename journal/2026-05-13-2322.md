# Journal 2026-05-13 23:22

## Summary

We reduced context pressure by splitting large `PTTViewModel` extensions into smaller files along real behavioral seams, then kept each split honest by rerunning a focused Swift test after every extraction.

## Problem

- Large `PTTViewModel+...` files were being read whole into context, which made debugging and iterative work expensive.
- The biggest files mixed unrelated concerns: transmit startup timing, direct QUIC identity/probe config, backend lifecycle/bootstrap, backend sync, and wake/signal handling.
- The concrete diagnostic pattern was the refactor repeatedly surfacing access-scope and exhaustiveness issues at the new file boundaries.

## Design formulation

- Use extension boundaries as the splitting unit, but only when the extracted chunk has a coherent responsibility.
- Keep the proof loop narrow: one mechanical extraction, one focused Swift test, fix only the symbols the split exposes.
- Prefer file-internal access over `private` when two same-type extensions genuinely need to share a helper across files.

## What changed

- Split `Turbo/PTTViewModel+Transmit.swift` into smaller transmit-focused files:
  - `Turbo/PTTViewModel+TransmitTiming.swift`
  - `Turbo/PTTViewModel+TransmitAudioSend.swift`
  - `Turbo/PTTViewModel+AudioSessionAndWake.swift`
- Split `Turbo/PTTViewModel+BackendCommands.swift` by invite/wait helpers.
- Split `Turbo/PTTViewModel+BackendLifecycle.swift` into lifecycle and identity/session files:
  - `Turbo/PTTViewModel+BackendLifecycleIdentityAndSession.swift`
- Split `Turbo/PTTViewModel+DirectQuic.swift` into identity/probe vs signaling/path handling:
  - `Turbo/PTTViewModel+DirectQuicIdentityAndProbe.swift`
- Split `Turbo/PTTViewModel+BackendSync.swift` into:
  - `Turbo/PTTViewModel+BackendSyncReceive.swift`
  - `Turbo/PTTViewModel+BackendSyncTransportFaultsAndSignals.swift`
- Fixed a real compiler exhaustiveness issue in `BackendSync.swift` by adding `.blockedByOtherSession` to the phase switch.
- Widened helpers to file-internal where new file boundaries required it.

## What worked

- The focused proof command stayed stable across all these refactors:
  - `just swift-test-target transmitStartupTimingSummaryIncludesFirstAudioStages`
- Each split reduced the size of the file without changing behavior.
- When the compiler failed, it gave direct evidence of the missing boundary contract instead of hiding the problem.

## What not to repeat

- Don’t split a file just because it is large if the extracted chunk still depends on too many `private` helpers.
- Don’t stop at one successful extraction if the next chunk is in the same locality and the proof loop is still green.
- Don’t let a split leave the codebase with one-off access hacks; fix the seam, not the symptom.

## Next
