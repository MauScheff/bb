# Journal 2026-06-01 12:46

## Summary

The self-hosted Compute Engine deployment path is implemented and deployed to a separate VM. The old relay VM remains untouched, while the new VM runs Postgres, Redis, and the Rust runtime with compiled Unison kernel artifacts.

## Problem

- Cloud Run was rejected as the first deployment target because Turbo needs TCP/UDP relay ownership on `443` and predictable realtime behavior.
- The existing relay VM must remain available while the new self-hosted backend is proved.
- Local UCM is useful for development, but the deployed runtime should use compiled Unison artifacts instead of a live codebase.

## Design formulation

- Use one new Compute Engine VM for early staging and low-volume production.
- Keep services separated by container: runtime, relay, Postgres, Redis.
- Keep Postgres as durable truth; Redis is only for ephemeral presence, routing, pub/sub, and short-lived coordination.
- Package compiled `.uc` kernel artifacts into the runtime image and run them with `ucm run.compiled`.
- Start only `postgres`, `redis`, and `runtime` until DNS, TLS, and relay `443` ownership are explicit.

## What changed

- Added VM deployment infrastructure under `infra/vm/`.
- Added kernel compilation tooling that emits deployable `.uc` artifacts.
- Added a GCE deploy script that builds a narrow release archive, installs Docker on the VM if needed, preserves remote credentials and volumes, and starts Docker Compose services.
- Added `just` recipes for kernel compilation, deploy dry-run, runtime deploy, and relay-profile deploy.
- Updated the Rust runtime kernel command runner to support compiled artifact mode.
- Documented the Compute Engine lane in `UNISON_KERNEL_RUST_RUNTIME.md` and `infra/vm/README.md`.

## What worked

- The new VM `turbo-self-hosted-1` is live in `beep-beep-495919/europe-west6-a`.
- Static IP `34.158.24.229` serves runtime health on port `8091`.
- Docker Compose reports Postgres and Redis healthy and the runtime running.
- The existing relay VM `turbo-relay-1` remains untouched.

## What not to repeat

- Do not start the Docker relay profile while another relay owns TCP/UDP `443`.
- Do not make the VM depend on a live Unison codebase for serving.
- Do not treat Redis as durable truth.
- Do not put Cloud Run back into the primary deploy path unless the transport requirements change.
