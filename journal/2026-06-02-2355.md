# Journal 2026-06-02 23:55

## Summary

Simulator fuzz seed `20260736` exposed a stale backend projection that could revive local transmit evidence while an explicit leave was already in flight. The fix is to make the leave-in-flight classification cover every projection derived from the stale refresh, not only joined-conversation reconciliation.

## Problem

- Fuzz artifact: `/tmp/turbo-scenario-fuzz/20260602-234522-8e939a81/seed-20260736`.
- Failed invariant: `engine.local_transmit_observation_requires_joined_conversation`.
- Strict diagnostics reported: `backend observed local transmit without a joined conversation`.
- Engine trace extraction and replay passed, so the engine reducer was not the owner.
- The app ignored a stale backend joined-conversation refresh during leave-in-flight, but still converted the same refresh's backend `self-transmitting` projection into `localTransmitObserved`.

## Design formulation

- Owner: app backend-sync projection guard at the engine boundary.
- Rule: once a contact has leave-in-flight evidence, a channel refresh may not publish backend local-transmit evidence for that contact into the engine.
- Reason: stale-refresh classification must be monotonic across all derived facts. A refresh cannot be too stale to reopen conversation membership but still fresh enough to assert local transmission.

## What changed

- `shouldAcceptBackendLocalTransmitProjection` now accepts a `leaveInFlight` flag and rejects backend local-transmit projection while leave is in flight.
- Channel refresh sync now passes both the captured `leaveWasInFlight` value and current pending leave status into the projection guard.
- Added a focused Swift regression proving backend `self-transmitting` does not revive local transmit during leave-in-flight.

## What worked

- The simulator replay for seed `20260736` is now strict-diagnostics green.
- The engine trace replay for the same artifact remains green with no invariant IDs or mismatches.
- The fix blocks the stale evidence at the app/backend event bridge before it becomes engine input, which addresses the source of the contradiction instead of masking the diagnostic.

## What not to repeat

- Do not treat stale backend refresh handling as a single-field filter. If one branch rejects joined-conversation evidence because leave is in flight, sibling projections from the same refresh need the same freshness boundary.
- Do not ask for device verification for this bug class. The failure is represented by simulator fuzz, strict diagnostics, and engine trace replay.
