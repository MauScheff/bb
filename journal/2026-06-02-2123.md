# Journal 2026-06-02 21:23

## Summary

Simulator fuzz seed `20260613` showed that a successful self-healing leave repair was being reported as a current invariant violation. The fix is to keep the repair behavior but emit it as state repair telemetry instead of failing strict diagnostics.

## Problem

- `just reliability-fuzz-local-overnight 20260611 500` stopped at simulator seed `20260613`.
- The scenario and engine trace replay passed, but strict diagnostics failed with `selected.stale_joined_refresh_during_leave`.
- The diagnostic phrase was `Ignored backend channel refresh that preserved joined membership while explicit leave was in flight`.

## Design formulation

- Ownership is app/backend-sync projection, not engine: the engine replay ended in valid `disconnecting` state with no mismatches.
- A delayed backend channel refresh may still show local joined membership after local Device PTT evidence has cleared during explicit leave.
- When the app ignores that refresh, clears local projection, clears receiver-readiness cache, and syncs the engine to disconnect, the invariant is protected. That event is repair evidence, not a current violation.

## What changed

- Reclassified the stale joined refresh leave-race event from `recordInvariantViolation` to `.state` diagnostics with `repairDecision=ignored-stale-joined-refresh`.
- Added `staleJoinedChannelRefreshDuringLeaveRecordsRepairTelemetryNotInvariantViolation`.
- Fixed `tools/scripts/test_merged_diagnostics.py` drift so production replay fixtures resolve from `shared/fixtures/production_replay` and intake summaries pass the newer audio-corpus arguments.

## What worked

- The exact replay artifact flipped from `strictDiagnosticsExitCode=2` to `0`.
- Strict diagnostics still fails real current violations; only the self-healed event was reclassified.

## What not to repeat

- Do not encode successful repair paths as current invariant violations. That turns a resilient state-machine edge into a flaky reliability gate failure.
- Do not relax the parser to ignore the invariant globally; the source event classification was wrong, not the strict diagnostics rule.
