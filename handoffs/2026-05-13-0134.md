## What is not working

- No active blocker is known at handoff time.
- Not yet fully proven by automated scenario matrix:
  - Ask from detail on either side.
  - Accept from home vs detail.
  - Ask, back out of detail, then receiver accepts.
  - Active call tap from home bypasses detail and returns directly to call screen.
- Merge diagnostics still needs a “current vs historical violations” separation.
- UI-state invariants exist conceptually, but not all detail/home/notification routes are encoded as simulator scenarios or invariant probes yet.
- Readiness latency is still being judged by physical-device timing. Avoid hard time-based correctness; prefer phase/order invariants and use timing only as performance telemetry.

## What to add back carefully

- Automatic diagnostics publish is debug default-on again for physical-device iteration. Reducer transition capture and state capture telemetry should still be turned on only for targeted debugging runs.
- Merge diagnostics should separate active/current violations from historical violations so old bad states do not confuse a clean run.
- Add UI state invariants for impossible screens:
  - call screen must not show plain “online” as the peer call state
  - selected/detail request state must not flap between requested and call screen
  - active call tap should bypass detail
  - minimizing or ending call should return to home, not leave stale detail stack state
- Add route tests/scenarios for detail/home/notification parity. The user should not have to manually discover every combination.
- Keep selected-contact prewarm idempotent. It is the main known fast-path win, but it must remain safe when repeated from contact detail, foreground notification, selected peer hint, and home list focus.
- Keep backend truth authoritative. Optimistic ask evidence is local pending-command evidence only, bounded by cooldown and replaced by backend state.

## Notes

- The important lesson from the 2-3 second ready path is that prewarm must start when contact intent becomes known, not when Ask/Accept is pressed.
- The important lesson from the freeze is that view/projection reads must remain pure. Synchronization belongs in explicit events, tasks, reducers, and command handlers.
- The important UI lesson is to keep the app Apple-native through structure and platform behavior, not decorative glass. Floating controls may use material; the primary conversation action should stay direct and unmistakable.
