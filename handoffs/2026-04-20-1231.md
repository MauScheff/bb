# Handoff 2026-04-20 12:31

## Summary

This handoff captures the architectural follow-up that should happen after the current physical-device PTT regressions are stabilized. The short-term loop is still "fix only the high-signal regressions that block device proof"; the next phase should be a structural state-model refactor, not more ad hoc patching.

## Current truth

- Apple PTT wake/receive has worked again on device after recent fixes, but follow-up regressions are still appearing in adjacent foreground/background handoff paths.
- The app currently models too much user-visible session state from mixed concerns: Apple session truth, backend membership, websocket transport, media warmup, and wake execution.
- Recent regressions show that transient backend/transport drift can still override durable session intent in UI/state derivation.

## What changed this session

- We explicitly identified the architectural issue as a modeling problem, not just a missing patch.
- We agreed on the next structural direction: separate durable session truth from recoverable control-plane state and transient execution state.

## What is not working

- The current state derivation still allows impossible states, such as active local transmit degrading into `wakeReady`/waiting-style projections when backend peer connectivity drifts.
- This makes the system feel patch-driven rather than invariant-driven.
- Hypothesis: the root issue is not that state machines are wrong, but that multiple distinct machines are being collapsed into one derived session state.

## Recommended next step

1. Finish the current device-blocking bugfix loop first.
2. Then refactor the state model into three explicit layers:
   `Session`: user-intent + Apple PTT session, durable until explicit disconnect or real system mismatch.
   `ControlPlane`: backend join, websocket, token upload, lease renewal; degradable and repairable under a live session.
   `Execution`: transmit / receive / wake activation / foreground prewarm; transient runtime activity.
3. Enforce hard invariants in reducer/domain code:
   `transmitting` must dominate `wakeReady` and generic waiting.
   backend drift must not auto-close a durable Apple session.
   `leaveChannel` is only legal for explicit disconnect or real system mismatch.
   websocket loss is degraded transport, not disconnected session.
