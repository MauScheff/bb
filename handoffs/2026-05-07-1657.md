## Overnight workflow

The fuzz runner is intentionally stop-on-first-failure. Treat each failure as one crisp counterexample:

1. Run overnight:

```bash
just serve-local
just simulator-fuzz-local-overnight 12345 500
```

2. If it fails, note the artifact directory printed by the command. It should contain the generated scenario, logs, merged diagnostics, metadata, and a replay script.

3. Replay exactly:

```bash
just simulator-fuzz-replay /tmp/turbo-scenario-fuzz/<run-id>/seed-<seed>
```

4. Shrink:

```bash
just simulator-fuzz-shrink /tmp/turbo-scenario-fuzz/<run-id>/seed-<seed>
```

5. After fixing, rerun:

```bash
just simulator-fuzz-replay /tmp/turbo-scenario-fuzz/<run-id>/seed-<seed>
just simulator-scenario-local "" http://localhost:8090/s/turbo
just simulator-fuzz-local 123 3
```

6. Then resume broader fuzzing with the same original overnight seed/count so the run can continue discovering the next independent bug.
